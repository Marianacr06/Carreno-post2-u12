# ADR-002: Factory + Strategy para seleccion de procesador

## Estado
Aceptado - 2026-03-23

## Contexto
Existen multiples tipos de pedido con algoritmos distintos. Un switch central viola OCP y complica extensiones.

## Decision
Se utiliza Strategy para cada tipo de pedido y Factory para seleccionar la estrategia en tiempo de ejecucion.

## Consecuencias
Positivas:
- Agregar un nuevo tipo es agregar una clase.
- Cada procesador se prueba de forma aislada.

Negativas:
- Mayor cantidad de clases comparado con un switch.
- La seleccion por mapa puede ser menos evidente para nuevos miembros.
