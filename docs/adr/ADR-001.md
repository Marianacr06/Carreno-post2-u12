# ADR-001: Arquitectura Hexagonal para aislar el dominio

## Estado
Aceptado - 2026-03-23

## Contexto
El sistema de pedidos debe soportar multiples tipos de procesamiento y canales de notificacion. El acoplamiento directo a infraestructura dificulta pruebas y cambios.

## Decision
Se adopta arquitectura hexagonal. El dominio define puertos (interfaces) y la infraestructura/adaptadores implementan esos puertos.

## Consecuencias
Positivas:
- Dominio testeable sin contenedor Spring.
- Cambios de persistencia sin tocar la logica de dominio.

Negativas:
- Aumenta la cantidad de clases e interfaces.
- Curva de aprendizaje mayor para el equipo.
