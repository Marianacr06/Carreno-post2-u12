# ADR-003: Spring Events (Observer) para notificaciones

## Estado
Aceptado — 2026-05-10

## Contexto
El servicio de pedidos necesita notificar a múltiples receptores (email, log, futuro SMS) cuando un pedido es procesado. El acoplamiento directo a cada canal hace frágil el servicio central.

## Decisión
Se usa Spring ApplicationEventPublisher (implementación de Observer) para publicar PedidoProcesadoEvent. Cada canal implementa un @EventListener independiente.

## Consecuencias
**Positivas:**
- Agregar un nuevo canal no requiere modificar FachadaPedidos.
- Los listeners son opcionales: si no hay suscriptores, el evento se ignora sin error.

**Negativas:**
- El orden de ejecución de listeners no está garantizado.
- El debugging es más complejo: hay que rastrear listeners para entender qué ocurre tras publicar el evento.
