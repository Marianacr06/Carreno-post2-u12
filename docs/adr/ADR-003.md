# ADR-003: Observer con Spring Events para notificaciones

## Estado
Aceptado - 2026-03-23

## Contexto
El servicio de pedidos debe notificar a multiples canales sin acoplarse a cada implementacion.

## Decision
Se usa `ApplicationEventPublisher` para publicar `PedidoProcesadoEvent`. Cada canal implementa un listener independiente.

## Consecuencias
Positivas:
- Nuevos canales se agregan sin modificar la fachada.
- Los listeners son opcionales.

Negativas:
- El orden de ejecucion no esta garantizado.
- El debugging requiere rastrear eventos y listeners.
